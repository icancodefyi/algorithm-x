# Password and multi-factor authentication

## Password requirements

- Minimum length **14 characters** for all corporate accounts unless a system enforces a higher bar.
- Passwords must not be reused across work and personal critical accounts.
- Default passwords and vendor-supplied credentials must be changed before production use.

## Multi-factor authentication (MFA)

- **MFA is mandatory** for corporate email, VPN, remote access, cloud admin consoles, and any system holding customer or regulated data.
- SMS-based OTP is discouraged where phishing-resistant methods (app, hardware token, passkey) are available.
- Recovery codes must be stored offline in a secure manner; sharing recovery codes is prohibited.

## Exceptions

Temporary MFA exceptions require **Security + manager approval** in writing and must include an expiry date (maximum 14 days unless extended by Security).

## Account lockout

Repeated failed authentication may trigger temporary lockout. Users must contact IT Helpdesk to unlock; self-service unlock is permitted only where IT has enabled it.
